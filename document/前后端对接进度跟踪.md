├───────────────────────────────┬──────────────────────────────────────┤
│  ⭐ GenAI 代理路径（主路径）    │  旧 ai-tools 路径（渐进淘汰）         │
│  /api/v1/genai/v1beta/*       │  /api/v1/ai-tools/*                 │

│  当前状态：SDK 直连 Google     │  当前状态：部分对接 + 多个 Bug         │
│  迁移方式：配置 Base URL + JWT │  状态：被 GenAI 代理取代              │
├───────────────────────────────┴──────────────────────────────────────┤

│  • 用户认证（登录/注册）                                               │

## 四、阻塞状态（已解决）

### ~~问题：缺少文件上传接口~~

**状态**：✅ **已解决**

**原问题**：图生视频、局部重绘、首尾帧视频需要 `image_url`，但前端用户产生的是 `File` 对象，后端缺少上传接口。

**解决方案**：后端已新增**预签名上传**接口：

| 接口 | 说明 |
|------|------|
| `POST /files/upload/presigned` | 获取单个预签名上传 URL |
| `POST /files/upload/presigned/batch` | 批量获取预签名 URL |
| `GET /files/upload/config` | 获取上传配置（允许类型、大小限制等） |

**前端使用流程**：
1. 调用 `POST /files/upload/presigned` 获取 `upload_url` 和 `access_url`
2. 用 `fetch(upload_url, { method: 'PUT', body: file })` 直传文件到存储服务
3. 使用 `access_url` 作为 `image_url` 传给 AI 工具接口

**可解锁的功能**（仅 ai-tools 旧路径）：
- 图生视频 `image-to-video` — 1 个 URL
- 局部重绘 `inpaint` — 2 个 URL（原图 + 蒙版）
- 首尾帧视频 `frame-to-video` — 2 个 URL（起始帧 + 结束帧）

**待办**：前端需在 `lib/api/files.ts` 新增预签名上传封装函数。
