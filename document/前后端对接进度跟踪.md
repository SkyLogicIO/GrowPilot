## 三、被阻塞（需后端配合）

### 问题：缺少文件上传接口

**影响范围**：图生视频、局部重绘、首尾帧视频

**原因**：上述三个后端接口均需要 `image_url`（URL 字符串），但前端用户操作产生的是 `File` 对象。后端当前没有文件上传（multipart）接口，前端无法将 File 转为 URL。

**阻塞链路**：
```
前端 File → [缺少上传接口] → image_url → 后端 AI 工具接口
```

**所需后端动作**：新增 `POST /api/v1/files/upload` 文件上传接口（支持 multipart/form-data），返回可访问的文件 URL。

**打通后可解锁**：
- 图生视频 `image-to-video` — 需要 1 个 URL
- 局部重绘 `inpaint` — 需要 2 个 URL（原图 + 蒙版）
- 首尾帧视频 `frame-to-video` — 需要 2 个 URL（起始帧 + 结束帧）

| # | 功能 | SDK 函数 | 前端文件 | 模型 |

**切换逻辑**：Art Studio / Video Factory 在用户上传参考图时，设置 `useBackendRef.current = false`，后续请求走 SDK 直连路径。

**已知限制**：
- Veo 3.1 有地区限制（中国大陆 IP 不可用），图生视频会报 `User location is not supported`
- API Key 暴露在 localStorage，存在安全隐患
- 不经过后端计费，无法追踪用量
