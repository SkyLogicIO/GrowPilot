## 一、后端服务架构概要

### 1.1 架构图

```
浏览器 ──► Go manage-server (:10086) ──► Python ai-server (:10087) ──► Google Gemini / Imagen / Veo
                │                              │
           MySQL (growpilot_business)     MySQL (growpilot_ai)
           JWT 认证 / 用户 / 任务            AI 策略 / 应用 / 用量 / Agent
```

### 1.2 两条对接路径

前端与后端 AI 能力的对接有两条路径：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         前端（Next.js 16.2）                          │
├──────────────────────────────────┬──────────────────────────────────┤
│  ⭐ GenAI 代理路径（推荐）         │  旧 ai-tools 路径（渐进淘汰）      │
│  /api/v1/genai/v1beta/*          │  /api/v1/ai-tools/*              │
│  对标 Google Gemini API          │  自定义异步任务接口               │
│  SDK 直连，配置 Base URL 即可     │  需要轮询获取结果                │
├──────────────────────────────────┼──────────────────────────────────┤
│  ✅ 文生图 (Imagen)               │  文生图（同步返回）               │
│  ✅ 图生图 (Gemini)               │  图生图（路由未注册）             │
│  ✅ 图片编辑 (Gemini)             │  局部重绘                        │
│  ✅ 文生视频 (Veo)                │  文生视频                        │
│  ✅ 图生视频 (Veo)                │  图生视频                        │
│  ✅ 多模态对话                    │  —                              │
│  ✅ 流式输出                      │  —                              │
│  ✅ 文本嵌入                      │  —                              │
└──────────────┬───────────────────┴──────────────┬───────────────────┘
               │                                  │
         Go 网关转发（透明代理）              Go 网关 → Python AI 层
         请求格式 = Google API 格式          自定义格式 + Go 验证器
```

**核心结论**：**GenAI 代理是主路径**。前端 `lib/ai.ts` 已在使用 `@google/genai` SDK，只需配置 Base URL 和 JWT header，所有 AI 功能（文生图、图生图、视频、对话）即可走后端，**业务代码无需改动**。

### 1.3 服务职责

| 服务 | 技术栈 | 职责 | 地址 |
|------|--------|------|------|
| **manage-server** | Go (Gin + GORM) | 对外 API 网关、JWT 认证、用户管理、**GenAI 代理**、Agent 代理、ai-tools 编排、文件服务 | `http://35.240.178.148:10086` |
| **ai-server** | Python (FastAPI + SQLAlchemy) | AI 模型调用、策略路由、应用/API Key 管理、用量统计、Agent 工作流引擎 | 内部服务，前端不直接访问 |

两服务之间通过 `X-Internal-API-Key` 头认证，**前端只需与 manage-server 通信**。

## 二、统一响应格式

### 2.1 自定义接口（ai-tools、auth、files 等）

后端自定义接口遵循统一格式：

```jsonc
// 成功
{ "code": 0, "message": "success", "data": { ... } }

// 失败
{ "code": -1, "message": "错误描述" }
```

HTTP 状态码：`200` 成功 / `201` 创建成功 / `400` 参数错误 / `401` 未认证 / `500` 服务错误。

**字段命名规范**：后端自定义接口全部使用 **snake_case**（如 `model_name`、`task_id`）。

### 2.2 GenAI 代理接口

GenAI 代理**直接透传 Google API 格式**，不使用上述 `{ code, message, data }` 包装。响应格式与 Google Gemini API 完全一致（参见第四节）。

## 三、认证体系

### 3.1 后端认证接口

#### 注册 `POST /api/v1/auth/register`

请求体：

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `email` | string | 是 | email 格式 |
| `password` | string | 是 | 6–100 字符 |
| `username` | string | 否 | 用户名 |

#### 登录 `POST /api/v1/auth/login`

请求体：

| 字段 | 类型 | 必填 |
|------|------|------|
| `username` | string | 是 |
| `password` | string | 是 |

> **注意**：后端当前使用 `username` 字段登录（非 `email`），前端 `auth.ts` 中的定义与后端一致。

#### 注册/登录统一成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "<JWT>",
    "expires_in": 86400,
    "user": {
      "id": 1,
      "email": "test@example.com",
      "username": "testuser",
      "user_type": 0,
      "credits": 100
    }
  }
}
```

> **实际字段说明**（已与后端源码 `auth_service.go` L29-54 核实）：
> - 响应 token 字段为 `access_token`（非 `token`）
> - `expires_in` 存在，值为 86400
> - `user` 对象包含 `user_type`（uint8）和 `credits`（int32）
> - 登录请求字段为 `username`（非 `email`）

#### 获取当前用户 `GET /api/v1/auth/me`

需要 `Authorization: Bearer <token>` 头。

## 四、GenAI 代理 — 核心对接方式

GenAI 代理是前端对接后端 AI 能力的**主要路径**。后端在 `/api/v1/genai/v1beta/` 下完全对标 Google Gemini API，Go 网关透明转发请求到 Google，同时处理 JWT 认证和用量统计。

### 4.1 配置方式（2 步）

前端 `lib/ai.ts` 已在使用 `@google/genai` SDK（v1.46.0），只需以下两步即可让所有 AI 请求走后端：

**第 1 步：设置 Base URL**

```typescript
import { setDefaultBaseUrls } from "@google/genai";

// 应用启动时执行一次
setDefaultBaseUrls({
  geminiUrl: 'http://35.240.178.148:10086/api/v1/genai/v1beta'
});
```

**第 2 步：创建 AI 实例时传入 JWT**

```typescript
import { GoogleGenAI } from "@google/genai";

const token = localStorage.getItem("access_token"); // 登录后获取的 JWT

const ai = new GoogleGenAI({
  apiKey: 'dummy',  // 后端不使用此 key，JWT 验证即可
  httpOptions: {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
});
```

**配置完成后，业务代码完全不用改**。`lib/ai.ts` 中所有函数（`generateImage`、`editImage`、`editImages`、`generateVideo`、`chatWithGemini`、`chatWithMedia`）创建的 `new GoogleGenAI({ apiKey })` 实例，只需加上 `httpOptions` 参数即可走后端。

### 4.2 覆盖范围

GenAI 代理覆盖 `lib/ai.ts` 的**全部功能**：

| lib/ai.ts 函数 | 功能 | SDK 调用 | 后端代理路径 |
|---------------|------|----------|------------|
| `generateImage()` 无参考图 | 文生图 | `ai.models.generateImages()` | `POST /genai/v1beta/models/{model}:predict` |
| `generateImage()` 有参考图 | 图生图 | `ai.models.generateContent()` | `POST /genai/v1beta/models/{model}:generateContent` |
| `editImage()` | 图片编辑（背景替换等） | `ai.models.generateContent()` | `POST /genai/v1beta/models/{model}:generateContent` |
| `editImages()` | 多图编辑 | `ai.models.generateContent()` | `POST /genai/v1beta/models/{model}:generateContent` |
| `generateVideo()` | 文生视频 / 图生视频 | `ai.models.generateVideos()` | `POST /genai/v1beta/models/{model}:predictLongRunning` |
| `chatWithGemini()` | 文本对话 | `ai.models.generateContent()` | `POST /genai/v1beta/models/{model}:generateContent` |
| `chatWithMedia()` | 多模态对话 + 图片生成 | 组合调用 | 组合调用 |

### 4.3 代理接口详情

以下接口均需 JWT 认证，Base path: `/api/v1/genai/v1beta`。

> **格式说明**：请求和响应格式与 Google Gemini API 完全一致，后端透明转发。字段命名使用 **camelCase**（Google API 规范）。

#### 生成内容（非流式） `POST /models/{model}:generateContent`

路径参数：`model` — 模型名称（如 `gemini-2.5-flash`、`gemini-2.5-flash-image`）

请求体：

```json
{
  "contents": [
    {
      "role": "user",
      "parts": [{ "text": "Hello" }]
    }
  ],
  "systemInstruction": {
    "parts": [{ "text": "You are a helpful assistant." }],
    "role": "user"
  },
  "generationConfig": {
    "temperature": 0.7,
    "topP": 0.9,
    "topK": 40,
    "maxOutputTokens": 2048,
    "responseModalities": ["TEXT", "IMAGE"]
  },
  "safetySettings": [...]
}
```

> 支持多模态：`parts` 中可包含 `{ "inlineData": { "mimeType": "image/png", "data": "<base64>" } }`。

响应（直接透传 Google API 格式）：

```json
{
  "candidates": [
    {
      "content": { "parts": [{ "text": "AI 回复" }], "role": "model" },
      "finishReason": "STOP",
      "index": 0
    }
  ],
  "usageMetadata": {
    "promptTokenCount": 10,
    "candidatesTokenCount": 20,
    "totalTokenCount": 30
  }
}
```

#### 生成内容（流式） `POST /models/{model}:streamGenerateContent`

请求体同上。响应格式为 `text/event-stream`（SSE），每个事件格式：

```
data: {"candidates": [{"content": {"parts": [{"text": "..."}]}}]}
```

#### 生成图片 `POST /models/{model}:predict`

用于 Imagen 模型（如 `imagen-4.0-generate-001`）。

请求体：

```json
{
  "instances": [{ "prompt": "A cute cat" }],
  "parameters": {
    "sampleCount": 1,
    "aspectRatio": "1:1"
  }
}
```

响应包含 `predictions` 数组，每项有 `bytesBase64Encoded` 字段。

#### 生成视频 `POST /models/{model}:predictLongRunning`

用于 Veo 模型（如 `veo-3.1-fast-generate-preview`）。

请求体：

```json
{
  "instances": [{ "prompt": "A cat walking" }],
  "parameters": {
    "aspectRatio": "16:9",
    "sampleCount": 1,
    "durationSeconds": 8,
    "resolution": "720p"
  }
}
```

返回 LRO（Long-Running Operation），后续通过 `GET /models/{model}:operations/{operationId}` 轮询状态。

#### 计数 Tokens `POST /models/{model}:countTokens`

请求体：`{ "contents": [{ "parts": [{ "text": "..." }] }] }`

响应：`{ "totalTokens": 42 }`

#### 嵌入内容 `POST /models/{model}:embedContent`

路径参数：`model` — 默认 `embedding-001`

请求体：

```json
{
  "content": { "parts": [{ "text": "要嵌入的文本" }] },
  "taskType": "RETRIEVAL_DOCUMENT"
}
```

响应：`{ "embedding": { "values": [0.1, 0.2, ...] } }`

#### 模型列表 `GET /models`

响应：

```json
{
  "models": [
    {
      "name": "models/gemini-2.0-flash",
      "displayName": "Gemini 2.0 Flash",
      "description": "...",
      "baseModelId": "gemini-2.0-flash",
      "version": "001",
      "supportedGenerationMethods": ["generateContent", "streamGenerateContent"]
    }
  ]
}
```

#### 模型详情 `GET /models/{model}`

响应结构同模型列表中的单个模型。

### 4.4 SDK 配置注意事项

1. **Base URL 必须包含 `/v1beta`**：
   - ✅ `http://35.240.178.148:10086/api/v1/genai/v1beta`
   - ❌ `http://35.240.178.148:10086/api/v1/genai`（缺少 v1beta）

2. **API Key 可以是任意值**：后端通过 JWT 验证身份，不使用 API Key。推荐使用 `'dummy'` 或空字符串。

3. **JWT Token 必须有效**：每个请求的 `Authorization: Bearer <token>` 头中的 token 必须是登录后获取的有效 JWT。

4. **`setDefaultBaseUrls` 只需调用一次**：建议在应用入口文件（如 `layout.tsx` 或专用初始化模块）中调用。

## 九、文件服务接口

Base path: `/api/v1/files`，均需 JWT。

### 9.1 上传配置 `GET /files/upload/config`

响应 `data`：

```json
{
  "storage_type": "gcs",
  "max_file_size": 104857600,
  "allowed_types": ["image/jpeg", "image/png", "image/gif", "image/webp", "video/mp4", "video/mov", "video/avi", "application/pdf"],
  "chunk_upload": true,
  "presigned_url": true,
  "default_expires_in": 3600
}
```

### 9.2 预签名上传 URL（单个） `POST /files/upload/presigned`

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `filename` | string | 是 | 文件名 |
| `content_type` | string | 是 | MIME 类型 |
| `folder` | string | 否 | 文件夹，默认 `uploads` |
| `path` | string | 否 | 自定义路径 |
| `expires_in` | number | 否 | 过期秒数，默认 3600 |

响应 `data`：

```json
{
  "upload_url": "https://storage.googleapis.com/...?signature=...",
  "access_url": "https://storage.googleapis.com/bucket/key",
  "bucket": "growpilot",
  "key": "uploads/image_001.png",
  "storage_type": "gcs",
  "method": "PUT",
  "headers": { "Content-Type": "image/png" },
  "expires_at": "2026-04-01T13:00:00Z"
}
```

**使用流程**：
1. 调用此接口获取 `upload_url`
2. 前端用 `fetch(upload_url, { method: 'PUT', body: file, headers })` 直传文件
3. 上传成功后，使用 `access_url` 作为文件 URL 传给 AI 工具接口

### 9.3 预签名上传 URL（批量） `POST /files/upload/presigned/batch`

请求体：

```json
{
  "files": [
    { "filename": "image1.png", "content_type": "image/png" },
    { "filename": "image2.png", "content_type": "image/png" }
  ],
  "folder": "uploads",
  "expires_in": 3600
}
```

响应 `data`：

```json
{
  "urls": [
    { "filename": "image1.png", "upload_url": "...", "access_url": "...", "key": "...", "method": "PUT", "headers": {...} },
    { "filename": "image2.png", "upload_url": "...", "access_url": "...", "key": "...", "method": "PUT", "headers": {...} }
  ],
  "count": 2
}
```

### 9.4 生成访问 URL `POST /files/generate-url`

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `bucket` | string | 是 | 存储桶名 |
| `key` | string | 是 | 文件路径/键 |
| `storage_type` | string | 是 | `local` / `oss` / `s3` |

响应 `data`：

```json

  "url": "https://...",
  "storage_type": "oss",
  "bucket": "growpilot",
  "key": "outputs/img_001.png"
}
```

### 9.5 批量生成 `POST /files/batch-generate-url`

请求体为数组，单项结构同上。失败项会被跳过。

### 9.6 解析元数据 `GET /files/parse-metadata?url=<encoded_url>`

从 URL 反解析出 `bucket`、`key`、`storage_type`。

## 十、前端架构改造建议

### 10.1 `lib/api/` 模块现状

```
lib/api/
  ├── client.ts       ✅ 统一 fetch 封装（baseURL、JWT 注入、401 拦截、错误处理）
  ├── types.ts        ✅ 认证响应结构与后端一致，无需修改
  ├── auth.ts         ✅ 登录/注册请求字段与后端一致，无需修改
  ├── ai-tools.ts     ⚠️ 旧路径封装，保留兼容，新功能不必新增
  ├── gemini.ts       ✅ 旧版 Gemini 接口，已有消费方（营销助手）
  └── files.ts        ✅ 预签名上传封装已完成
```

### 10.2 `lib/ai.ts` 迁移策略（配置 Base URL + JWT）

`lib/ai.ts` 当前直连 Google GenAI（需要用户配置 `gemini_api_key`）。迁移方案**极其简单**——只需修改实例创建方式，业务代码不用改：

| 步骤 | 改动 | 说明 |
|------|------|------|
| **1. 全局设置 Base URL** | 应用入口加 3 行代码 | `setDefaultBaseUrls({ geminiUrl: '...' })` |
| **2. 修改 AI 实例创建** | 每处 `new GoogleGenAI({ apiKey })` 加 `httpOptions` | 注入 JWT header |
| **3. 移除 localStorage API Key** | 删除 `getApiKey()` 和设置页面入口 | API Key 由后端管理，前端不感知 |

**关键优势**：
- **零业务逻辑改动**：`generateImage()`、`editImage()`、`generateVideo()` 等函数签名和调用方式完全不变
- **移除 localStorage 依赖**：不再需要 `gemini_api_key`，用户无需配置
- **统一认证**：所有 AI 请求走 JWT，与后端其他接口一致
- **后端计费**：AI 用量可被后端追踪

### 10.3 数据持久化迁移

| 当前方案 | 迁移后方案 |
|---------|-----------|
| `gemini_api_key`（localStorage） | 后端统一管理，前端通过 JWT 认证 |
| `growpilot_projects`（localStorage） | 后端任务系统 + 文件服务 |
| `growpilot_user_profile`（Demo 数据） | 后端 `/auth/me` 真实用户信息 |
