# GrowPilot 前后端对接指导文档

> **后端服务地址**：`http://35.240.178.148:10086`
>
> **最后更新**：2026-04-01
>
> 本文档基于后端仓库 `growpliot-server`（Go manage-server + Python ai-server）的当前实现，为前端项目 `growpilot-v2` 的后续对接工作提供系统性指导。随着后端功能完善，本文档将持续更新。

---

## 一、后端服务架构概要

```
浏览器 ──► Nginx ──► Go manage-server (:10086) ──► Python ai-server (:10087) ──► Google Gemini / Imagen / Veo
                          │                              │
                     MySQL (growpilot_business)     MySQL (growpilot_ai)
                     JWT 认证 / 用户 / 任务            AI 策略 / 应用 / 用量
```

| 服务 | 技术栈 | 职责 | 地址 |
|------|--------|------|------|
| **manage-server** | Go (Gin + GORM) | 对外 API 网关、JWT 认证、用户管理、任务编排、文件 URL 服务 | `http://35.240.178.148:10086` |
| **ai-server** | Python (FastAPI + SQLAlchemy) | AI 模型调用、策略路由、应用/API Key 管理、用量统计 | 内部服务，前端不直接访问 |

两服务之间通过 `X-Internal-API-Key` 头认证，**前端只需与 manage-server 通信**。

## 二、统一响应格式

后端所有接口遵循统一格式：

```jsonc
// 成功
{ "code": 0, "message": "success", "data": { ... } }

// 失败
{ "code": -1, "message": "错误描述" }
```

HTTP 状态码：`200` 成功 / `201` 创建成功 / `400` 参数错误 / `401` 未认证 / `500` 服务错误。

**字段命名规范**：后端全部使用 **snake_case**（如 `model_name`、`task_id`），文档中如有 camelCase 示例与代码不一致，以 snake_case 为准。

## 三、认证体系

### 3.1 当前前端现状

- `Login.tsx` 使用 `setTimeout` 模拟登录，写入 `localStorage.setItem("access_token", "mock-token")`
- Dashboard 布局仅检查 `access_token` 是否存在，不校验有效性
- 自定义事件体系已就绪：`growpilot:login` / `growpilot:logout` / `growpilot:unauthorized`

### 3.2 后端认证接口

#### 注册 `POST /api/v1/auth/register`

请求体：

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `username` | string | 是 | 3–50 字符 |
| `email` | string | 是 | email 格式 |
| `password` | string | 是 | 6–100 字符 |
| `phone` | string | 否 | 最长 20 字符 |

#### 登录 `POST /api/v1/auth/login`

请求体：

| 字段 | 类型 | 必填 |
|------|------|------|
| `username` | string | 是 |
| `password` | string | 是 |

#### 注册/登录统一成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "<JWT>",
    "expires_in": 86400,
    "token_type": "Bearer",
    "user": {
      "id": 1,
      "username": "testuser",
      "email": "test@example.com",
      "user_type": 1,
      "credits": 100
    }
  }
}
```

- `user_type`：1 普通 / 2 VIP / 3 企业
- `credits`：积分余额，新注册用户默认 100
- JWT 有效期 24 小时，暂无 refresh token 机制

#### 获取当前用户 `GET /api/v1/auth/me`

需要 `Authorization: Bearer <token>` 头。

### 3.3 前端改造方案

| 改造项 | 说明 |
|--------|------|
| **Login.tsx** | 将模拟逻辑替换为 `fetch` 真实调用 `/auth/login`；新增注册表单对接 `/register` |
| **Token 存储** | 登录成功后存储真实 JWT 到 `localStorage("access_token")`；同时存储 `user` 对象 |
| **请求拦截器** | 在 `lib/api/client.ts` 中封装统一 `request` 函数，自动附加 `Authorization: Bearer <token>` 头 |
| **401 处理** | API 返回 401 时 `dispatchEvent(new CustomEvent("growpilot:unauthorized"))`，触发已有登出逻辑 |
| **Token 过期** | 24h 后 token 过期，引导用户重新登录 |

### 3.4 用户数据模型对齐

| 后端字段 | 类型 | 前端当前对应 | 对接说明 |
|----------|------|-------------|----------|
| `id` | number | 无 | 新增，用于关联业务 |
| `username` | string | `growpilot_user.name` | 对齐 |
| `email` | string | `growpilot_user.email` | 对齐 |
| `user_type` | number | `growpilot_user_profile.membership` | 映射 1→普通 2→VIP 3→企业 |
| `credits` | number | `growpilot_user_profile.points` | 对齐，取消 Demo 数据 |

## 五、任务系统

后端采用**异步任务模型**，AI 工具接口创建任务后返回 `task_id`，前端需轮询查询结果。

### 5.1 任务状态流转

```
pending ──► processing ──► completed
                      └──► failed
                      └──► cancelled
```

### 5.2 查询任务 `GET /ai-tools/tasks/:task_id`

响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "abc123",
    "status": "completed",
    "progress": 100,
    "result_url": "https://storage.example.com/output.png",
    "error": null,
    "created_at": "2026-04-01T12:00:00Z",
    "completed_at": "2026-04-01T12:01:30Z"
  }
}
```

### 5.3 任务列表 `GET /ai-tools/tasks`

查询参数：`status`、`task_type`、`page`、`page_size`。

> ⚠️ **当前状态**：此接口返回占位数据，待后端实现。

### 5.4 取消任务 `POST /ai-tools/tasks/:task_id/cancel`

> ⚠️ **当前状态**：此接口未真正实现，待后端完善。

### 5.5 前端轮询建议

1. 调用 AI 工具接口获取 `task_id`
2. 每 3–5 秒轮询 `GET /ai-tools/tasks/:task_id`
3. 根据 `status` 更新 UI，`progress` 字段驱动进度条
4. `completed` → 取 `result_url` 展示结果
5. `failed` → 展示 `error` 信息
6. 建议封装 `useTaskPolling(taskId)` Hook 统一管理

## 八、前端架构改造建议

### 8.1 新增 `lib/api/` 模块

```
lib/api/
  ├── client.ts       # 统一 fetch 封装（baseURL、JWT 注入、401 拦截、错误处理）
  ├── auth.ts         # register / login / getMe
  ├── ai-tools.ts     # textToImage / textToVideo / imageToVideo / inpaint / getTask / listTasks
  ├── gemini.ts       # chat / generate / countTokens / getModels
  ├── files.ts        # generateUrl / batchGenerateUrl / parseMetadata
  └── types.ts        # 所有请求/响应 TypeScript 类型定义
```

### 8.2 统一请求客户端 `client.ts` 核心逻辑

```typescript
const BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL || "http://35.240.178.148:10086";

async function request<T>(endpoint: string, options?: RequestInit): Promise<T> {
  const token = localStorage.getItem("access_token");
  const headers: HeadersInit = {
    "Content-Type": "application/json",
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
    ...options?.headers,
  };

  const res = await fetch(`${BASE_URL}${endpoint}`, { ...options, headers });

  if (res.status === 401) {
    window.dispatchEvent(new CustomEvent("growpilot:unauthorized"));
    throw new Error("Unauthorized");
  }

  const json = await res.json();
  if (json.code !== 0) {
    throw new Error(json.message || "请求失败");
  }
  return json.data;
}
```

### 8.3 `lib/ai.ts` 渐进式迁移策略

`lib/ai.ts` 当前直连 Google GenAI，**不建议立即全部移除**：

| 阶段 | 策略 |
|------|------|
| **阶段 1（当下）** | 保留 `lib/ai.ts` 作为开发/演示用，新增 `lib/api/` 对接后端 |
| **阶段 2（后端稳定后）** | 各页面逐步切换到后端 API，可用环境变量或 feature flag 控制 |
| **阶段 3（全面对接）** | 移除 `gemini_api_key` 的 localStorage 依赖，API Key 仅在后端管理 |

### 8.4 数据持久化迁移

| 当前方案 | 对接后方案 |
|---------|-----------|
| `growpilot_projects`（localStorage） | 后端任务系统 + 文件服务 |
| `growpilot_user_profile`（Demo 数据） | 后端 `/auth/me` 真实用户信息 |
| `gemini_api_key`（localStorage） | 后端统一管理，前端无需感知 |

## 九、字段映射差异表

前端当前字段与后端期望字段的对照，对接时需注意转换：

| 功能 | 前端当前字段 | 后端期望字段 | 说明 |
|------|-------------|-------------|------|
| 认证 | `api_key`（前端传给 proxy） | JWT `Authorization` 头 | 移除前端显式传 key |
| 文生视频 | `duration`（string `"8"`） | `duration`（number） | 类型对齐 |
| 文生视频 | `resolution`（`"720p"`） | `width` + `height`（number） | 前端做分辨率→宽高转换 |
| 文生视频 | `aspect_ratio`（`"16:9"`） | 由 `width`/`height` 决定 | 前端根据比例计算 |
| 文生图 | `model` | `model_name` | 字段名不同 |
| 换脸 | `input_image` + `source_image`（File） | 需先上传获取 URL | 流程差异 |
| 项目数据 | `resultUrl`（data URL / blob） | `result_url`（HTTP URL） | 不再存 base64 |
