## 一、技术栈总览

| 维度 | 外包版 | V2 |
|------|--------|-----|
| 框架 | Next.js 16.2 + React 19 | Next.js 16.2 + React 19 |
| 样式 | Tailwind 3 + `tailwind.config.ts` | Tailwind 4 + CSS `@theme` |
| 构建 | Webpack（`--webpack` 标志） | Turbopack（默认） |
| 包管理 | npm | pnpm |
| AI SDK | `@google/generative-ai` 0.24 | `@google/genai` 1.46 |
| API 层 | `lib/api/` → Go 后端 `:8080` | `app/api/proxy/` → 直连 Google + 外部代理 |
| 认证 | JWT 完整流程（注册/登录/刷新） | Mock 登录 |
| 数据 | 后端 SQLite + API | 纯 localStorage |
| 设计 | 深色主题 `#0B0D10` | Soft Brutalism `#FAF9F4` |
| 前端语言 | 混合 `.jsx/.tsx` | 全 `.tsx` |
| Dev 端口 | 3010 | 3005 |

## 二、业务板块逐项对比

### 1. 仪表板首页 `/dashboard`

**外包版**

- 欢迎横幅（个性化问候 "欢迎回来, Anna"）
- 4 个统计卡片：本周创作数、资产容量、获得积分、算力消耗
- 4 个快捷入口：新建项目、添加素材、AI 视频工厂、AI 创作工具
- 最近 8 个项目网格（缩略图 + 状态徽章 + 时间戳）
- 数据来源：`authApi.getUserSummary()` 从后端获取 + `lib/demoProjects` 本地 demo

**V2**

- 欢迎横幅（个性化问候）
- 4 个统计卡片：本周创作数、资产容量、获得积分、算力消耗
- 4 个快捷入口：新建项目、添加素材、AI 视频工厂、AI 创作工具
- 最近 8 个项目网格（缩略图 + 状态徽章 + 时间戳）
- 数据来源：`lib/demoProjects` 本地 demo（与外包版 UI 结构一致）

**差异总结**

- UI 结构基本一致，核心差异在数据层：外包版从后端 API 获取统计数据，V2 使用本地 demo 数据
- V2 需要补充：后端数据对接（`getUserSummary()` API）

### 2. 营销助手 `/dashboard/marketing-assistant`

**外包版**

- **双模式切换**：对话模式 + 画布模式（`ModeSwitcher` 组件）
- 组件拆分为独立文件：
  - `ChatHeader.tsx` — 顶部工具栏
  - `ChatSidebar.tsx` — 会话历史侧栏
  - `ModeSwitcher.tsx` — 模式切换器
  - `CanvasView.tsx` — 画布视图
- 独立聊天组件：
  - `ChatInputBox.tsx` — 支持附件上传、模型选择、语言切换、模板库入口
  - `RoleBubble.tsx` — 用户/AI 消息气泡
- 工具预设：智能问答、产生文案、拍摄脚本、电商卖点、创作素材
- 推荐工具面板（可刷新轮换）
- AI 模型：`gemini-3-flash-preview`（旧 SDK）
- System Prompt：内联在组件中

**V2**

- 单一对话模式，无画布模式
- 所有 UI 内联在 `MarketingAssistantPageClient.tsx` 单文件
- 聊天输入框较简单，无附件/模型选择/模板库
- 同样的工具预设
- AI 模型：`gemini-2.5-flash`（新 SDK）
- 图片编辑已从 Imagen editImage（仅 Vertex AI）切换到 Gemini generateContent（普通 API Key 可用）
- System Prompt：独立文件 `lib/prompts/marketing-prompts.ts`，包含 5 套完整结构化 prompt（智能问答/产生文案/拍摄脚本/电商卖点/创作素材），每套含输出格式规范

**差异总结**

- 外包版组件拆分更合理，功能更丰富（画布模式、富输入框）
- V2 的 System Prompt 质量更高（独立文件、5 套完整结构化 prompt、输出格式规范，外包版仅内联简单提示）
- V2 的图片编辑技术路线更优（Gemini generateContent 通用 API vs 外包版 Imagen editImage 需 Vertex AI）
- V2 需要补充：画布模式、ChatInputBox 富工具栏、组件拆分

## 三、外包版有而 V2 缺失的关键功能

### 优先级高

1. **画布模式**（营销助手 Canvas View）— 外包版已完整实现，V2 完全缺失
2. **ChatInputBox 富工具栏**（附件上传 / 模型选择 / 语言切换 / 模板库）— 外包版拆为独立组件
3. **后端数据驱动的仪表板**（统计卡片 + 最近项目 + 用户摘要 API）
4. **后端认证**（注册 / 登录 / Token 刷新 — V2 为 Mock 登录，无后端认证）

### 优先级中

5. **项目后端 CRUD**（外包版通过 API，V2 仅 localStorage）
6. **积分 & 存储系统后端对接**（UI 骨架两版都有，但都未接入后端）
7. **组件模块化拆分**（营销助手的 ChatHeader/ChatSidebar/RoleBubble 等独立组件）

### 优先级低

8. **修改密码 / 锁定屏幕**（UserMenu 中的功能）
9. **InviteRewardButton / CoinsVipButton**（Header 中的积分相关 UI）

## 四、V2 的优势（外包版没有的）

1. **独立创作页面** — Art Studio + Video Factory 独立路由，体验更好
2. **换背景功能** — `BackgroundReplaceModal`（Gemini generateContent）
3. **Soft Brutalism 设计系统** — 独特视觉语言，brut-card/brut-btn/brut-tag 工具类
4. **新版 AI SDK** — `@google/genai` 1.46，支持 Imagen 4.0 + Veo 3.1
5. **图片编辑技术路线升级** — 从 Imagen editImage（需 Vertex AI）切换到 Gemini generateContent（普通 API Key 可用）
6. **全 TypeScript** — 零 `.jsx/.js` 残留
7. **结构化 System Prompt** — `lib/prompts/marketing-prompts.ts`，5 套完整 prompt（智能问答/产生文案/拍摄脚本/电商卖点/创作素材），每套含角色定义、输出格式规范、示例，远超外包版的内联简单提示
8. **客户端存储系统** — `lib/storage/` 带版本化 schema 和自动清理
9. **首页 6 模块对齐 MVP** — AI 视频工场、AI 绘画工作室、AI 智能编辑、AI 模特换装、AI 局部重绘、AI 营销文案（外包版旧版为爆款案例拆解、云端素材管家等）
10. **ComingSoon 占位机制** — 未上线功能的统一处理
11. **Turbopack** — 更快的开发构建
